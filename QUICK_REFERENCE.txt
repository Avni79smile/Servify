# SUBSCRIPTION FIX - QUICK REFERENCE CARD

Keep this open while applying the fix!

## THE PROBLEM (In 30 seconds)

```
User clicks "Complete Purchase"
         ↓
Toast says "Payment successful" ✅
         ↓
Database shows plan_id = NULL ❌

Why? auth.uid() was NULL in RLS policy
     (Supabase couldn't authenticate the insert)
```

## THE SOLUTION (3 Steps)

### Step 1: Run SQL (2 min)
```
1. Go to: https://app.supabase.com
2. Select your project
3. Click: SQL Editor
4. Click: New Query
5. Copy file: FIX_SUBSCRIPTION_RLS.sql
6. Paste into editor
7. Click: Run
8. ✅ Check: No red errors
```

### Step 2: Restart App (1 min)
```bash
# In terminal:
npm run dev
# Wait for: "Local: http://localhost:5173"
```

### Step 3: Test (2 min)
```
1. Open: http://localhost:5173
2. Login
3. Go to: Subscription Plans
4. Select: Family Basic or Pro
5. Click: Complete Purchase
6. Open: DevTools (F12)
7. Console tab
8. ✅ Should show: "Subscription saved successfully"
9. ✅ Open Supabase → Table Editor → user_subscriptions
10. ✅ See new row with plan_id (NOT NULL)
```

## IF IT FAILS

### See error: "Failed to save subscription"
```
1. Copy error from console (F12)
2. Check DEEP_DEBUG_ANALYSIS.md
3. Look up error code in table
4. Follow suggested fix
```

### See "Payment successful" but NO data in DB
```
1. Check console → any red text?
2. Run SQL test in Supabase:
   SELECT * FROM user_subscriptions 
   WHERE user_id = auth.uid();
3. Check it returns something (not empty)
```

### See "RLS policy" or "403" error
```
1. Run SQL fix again
2. Make sure you clicked "Run" (not just pasted)
3. Verify 4 policies exist:
   - subscribe_insert
   - subscribe_select
   - subscribe_update
   - subscribe_delete
4. Logout/login and retry
```
